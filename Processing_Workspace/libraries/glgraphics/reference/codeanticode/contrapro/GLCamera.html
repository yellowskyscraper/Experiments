Author:
pierre A perspective or orthographic camera.

A GLCamera defines some intrinsic parameters (fieldOfView(), position(), viewDirection(), upVector()...) and useful positioning tools that ease its placement (showEntireScene(), fitSphere(), lookAt()...). It exports its associated OpenGL projection and modelview matrices and can interactively be modified using the mouse.

computeModelViewMatrix

public void computeModelViewMatrix()
Computes the modelView matrix associated with the Camera's position() and orientation(). This matrix converts from the world coordinates system to the GLCamera coordinates system, so that coordinates can then be projected on screen using the projection matrix (see computeProjectionMatrix()). Use getModelViewMatrix() to retrieve this matrix. Note: You must call this method if your GLCamera is not associated with a GLGraphics and is used for offscreen computations (using (un)projectedCoordinatesOf() for instance). loadModelViewMatrix() does it otherwise.

computeProjectionMatrix

public void computeProjectionMatrix()
Computes the projection matrix associated with the GLCamera.

If type() is PERSPECTIVE, defines a GL_PROJECTION matrix similar to what would gluPerspective() do using the fieldOfView(), window aspectRatio(), zNear() and zFar() parameters.

If type() is ORTHOGRAPHIC, the projection matrix is as what glOrtho() would do. Frustum's width and height are set using getOrthoWidthHeight().

Both types use zNear() and zFar() to place clipping planes. These values are determined from sceneRadius() and sceneCenter() so that they best fit the scene size.

Use getProjectionMatrix() to retrieve this matrix. Overload loadProjectionMatrix() if you want your GLCamera to use an exotic projection matrix.

Note: You must call this method if your GLCamera is not associated with a GLGraphics and is used for offscreen computations (using (un)projectedCoordinatesOf() for instance). loadProjectionMatrix() does it otherwise.

convertClickToLine

public void convertClickToLine(Point pixel,
                               PVector orig,
                               PVector dir)
Gives the coefficients of a 3D half-line passing through the GLCamera eye and pixel (x,y).

The origin of the half line (eye position) is stored in orig, while dir contains the properly oriented and normalized direction of the half line.

x and y are expressed in Processing format (origin in the upper left corner). Use screenHeight() - y to convert to OpenGL units.

This method is useful for analytical intersection in a selection method.

distanceToSceneCenter

public float distanceToSceneCenter()
Returns the distance from the Camera center to sceneCenter(), projected along the Camera Z axis.

Used by zNear() and zFar() to optimize the Z range.

fieldOfView

public float fieldOfView()
Returns the vertical field of view of the GLCamera (in radians).

Value is set using setFieldOfView(float). Default value is pi/4 radians. This value is meaningless if the GLCamera type() is GLCamera.ORTHOGRAPHIC.

The field of view corresponds the one used in gluPerspective (see manual). It sets the Y (vertical) aperture of the GLCamera. The X (horizontal) angle is inferred from the window aspect ratio (see aspectRatio() and horizontalFieldOfView()).

Use setFOVToFitScene() to adapt the fieldOfView() to a given scene.

fitScreenRegion

public void fitScreenRegion(Rectangle rectangle)
Moves the Camera so that the rectangular screen region defined by rectangle (pixel units, with origin in the upper left corner) fits the screen.

The GLCamera is translated (its orientation() is unchanged) so that rectangle is entirely visible. Since the pixel coordinates only define a frustum in 3D, it's the intersection of this frustum with a plane (orthogonal to the viewDirection() and passing through the sceneCenter()) that is used to define the 3D rectangle that is eventually fitted.

fitSphere

public void fitSphere(PVector center,
                      float radius)
Moves the GLCamera so that the sphere defined by center, radius) is visible and fits the window.

The GLCamera is simply translated along its viewDirection() so that the sphere fits the screen. Its orientation() and its fieldOfView() are unchanged.

You should therefore orientate the GLCamera before you call this method.

focusDistance

public float focusDistance()
Returns the focus distance used by stereo display, expressed in OpenGL units.

This is the distance in the virtual world between the Camera and the plane where the horizontal stereo parallax is null (the stereo left and right images are superimposed).

This distance is the virtual world equivalent of the real-world physicalDistanceToScreen().

attention: This value is modified by GLGraphics.setSceneRadius(), setSceneRadius() and setFieldOfView(float). When one of these values is modified, focusDistance() is set to sceneRadius() / tan(fieldOfView()/2), which provides good results.

Returns the GLInteractiveCameraFrame attached to the GLCamera.

fromModelViewMatrix

public void fromModelViewMatrix(float[] modelViewMatrix)
Sets the GLCamera's position() and orientation() from an OpenGL ModelView matrix.

This enables a GLCamera initialisation from an other OpenGL application. modelView is a 16 float vector representing a valid OpenGL ModelView matrix, such as one can get using:

GLfloat mvm[16];
glGetFloatv(GL_MODELVIEW_MATRIX, mvm);
myCamera.fromModelViewMatrix(mvm);

After this method has been called, #g getModelViewMatrix() returns a matrix equivalent to modelView.

Only the orientation() and position() of the GLCamera are modified.

getFrustumPlanesCoefficients

public float[][] getFrustumPlanesCoefficients()
Returns the 6 plane equations of the GLCamera frustum.

The six 4-component vectors of coef respectively correspond to the left, right, near, far, top and bottom GLCamera frustum planes. Each vector holds a plane equation of the form:

a*x + b*y + c*z + d = 0

where a, b, c and d are the 4 components of each vector, in that order.

This format is compatible with the glClipPlane() function. One camera frustum plane can hence be applied in an other viewer to visualize the culling results:

// Retrieve place equations
float [][] coef = mainViewer.camera().getFrustumPlanesCoefficients();
// These two additional clipping planes (which must have been enabled)
// will reproduce the mainViewer's near and far clipping.
glClipPlane(GL_CLIP_PLANE0, coef[2]);
glClipPlane(GL_CLIP_PLANE1, coef[3]);

getModelViewMatrix

public float[] getModelViewMatrix(float[] m)
Fills m with the Camera modelView matrix values and returns it. If m is null (or not the correct size), a new array will be created.

First calls computeModelViewMatrix() to define the GLCamera modelView matrix.

Note that this matrix is usually not the one you would get from a glGetFloatv(GL_MODELVIEW_MATRIX, m). It converts from the world to the GLCamera coordinate system, but as soon as you modify the GL_MODELVIEW in your drawing code, the two matrices differ.

The result is an OpenGL 4x4 matrix, which is given in column-major order (see glMultMatrix documentation for details).

See Also:
getProjectionMatrix(float[]), fromModelViewMatrix(float[])

getOrthoWidthHeight

public float[] getOrthoWidthHeight()
Convenience function that simply returns getOrthoWidthHeight(new float[2]).

getOrthoWidthHeight

public float[] getOrthoWidthHeight(float[] target)
Fills in target with the halfWidth and halfHeight of the GLCamera orthographic frustum and returns it.

While holds halfWidth, target[1] holds halfHeight.

These values are only valid and used when the GLCamera is of type() ORTHOGRAPHIC. They are expressed in OpenGL units and are used by loadProjectionMatrix() to define the projection matrix using: glOrtho( -halfWidth, halfWidth, -halfHeight, halfHeight, zNear(), zFar() ).

These values are proportional to the GLCamera (z projected) distance to the revolveAroundPoint(). When zooming on the object, the GLCamera is translated forward \e and its frustum is narrowed, making the object appear bigger on screen, as intuitively expected.

Overload this method to change this behavior if desired.

getProjectionMatrix

public float[] getProjectionMatrix(float[] m)
Fills m with the GLCamera projection matrix values and returns it. If m is null (or not the correct size), a new array will created.

Calls computeProjectionMatrix() to define the GLCamera projection matrix.

This matrix only reflects the GLCamera's internal parameters and it may differ from the GL_PROJECTION matrix retrieved using glGetFloatv(GL_PROJECTION_MATRIX, m).

The result is an OpenGL 4x4 matrix, which is given in column-major order (see glMultMatrix documentation for details). See also getModelViewMatrix() and fromProjectionMatrix().

getViewport

public int[] getViewport(int[] viewport)
Fills viewport with the GLCamera OpenGL viewport and returns it. If viewport is null (or not the correct size), a new array will be created.

This method is mainly used in conjunction with gluProject, which requires such a viewport. Returned values are (0, screenHeight(), screenWidth(), - screenHeight()), so that the origin is located in the upper left corner of the window.

horizontalFieldOfView

public float horizontalFieldOfView()
Returns the horizontal field of view of the GLCamera (in radians).

Value is set using setHorizontalFieldOfView(float) or setFieldOfView(float). These values are always linked by: horizontalFieldOfView() = 2.0 * atan ( tan(fieldOfView()/2.0) * aspectRatio() ).

physicalDistanceToScreen

public float physicalDistanceToScreen()
Returns the physical distance between the user's eyes and the screen (in meters).

Default value is 0.5m.

Used by loadModelViewMatrixStereo() and loadProjectionMatrixStereo() for stereo display. Value is set using setPhysicalDistanceToScreen().

physicalDistanceToScreen() and focusDistance() represent the same distance. The first one is expressed in physical real world units, while the latter is expressed in OpenGL virtual world units. Use their ratio to convert distances between these worlds.

physicalScreenWidth

public float physicalScreenWidth()
Returns the physical screen width, in meters. Default value is 0.4m (average monitor).

Used for stereo display only (see loadModelViewMatrixStereo() and loadProjectionMatrixStereo()). Set using setPhysicalScreenWidth(float). See physicalDistanceToScreen() for reality center automatic configuration.

pixelGLRatio

public float pixelGLRatio(PVector position)
Returns the ratio between pixel and OpenGL units at position.

A line of n * pixelGLRatio() OpenGL units, located at position in the world coordinates system, will be projected with a length of n pixels on screen.

Use this method to scale objects so that they have a constant pixel size on screen. The following code will draw a 20 pixel line, starting at sceneCenter() and always directed along the screen vertical direction:

gl.glBegin(GL.GL_LINES);
gl.glVertex3fv(sceneCenter());
gl.glVertex3fv(sceneCenter() + 20 * pixelGLRatio(sceneCenter()) * camera().upVector());
gl.glEnd();

position

public final PVector position()
Returns the GLCamera position (the eye), defined in the world coordinate system.

Use setPosition(PVector) to set the Camera position. Other convenient methods are showEntireScene() or fitSphere(). Actually returns GLFrame.position().

This position corresponds to the projection center of a GLCamera.PERSPECTIVE camera. It is not located in the image plane, which is at a zNear() distance ahead.

projectedCoordinatesOf

public final PVector projectedCoordinatesOf(PVector src,
                                            GLFrame frame)
Returns the screen projected coordinates of a point src defined in the frame coordinate system.

When frame is null, src is expressed in the world coordinate system. See projectedCoordinatesOf(PVector).

The x and y coordinates of the returned PVector are expressed in pixel, (0,0) being the upper left corner of the window. The z coordinate ranges between 0.0 (near plane) and 1.0 (excluded, far plane). See the gluProject man page for details.

unprojectedCoordinatesOf() performs the inverse transformation. Attention: This method only uses the intrinsic GLCamera parameters (see getModelViewMatrix(), getProjectionMatrix() and getViewport()) and is completely independent of the OpenGL GL_MODELVIEW, GL_PROJECTION and viewport matrices. You can hence define a virtual GLCamera and use this method to compute projections out of a classical rendering context.

sceneCenter

public PVector sceneCenter()
Returns the position of the scene center, defined in the world coordinate system.

The scene observed by the GLCamera should be roughly centered on this position, and included in a sceneRadius() sphere. This approximate description of the scene permits a zNear() and zFar() clipping planes definition, and allows convenient positioning methods such as showEntireScene().

Note that GLGraphics.sceneCenter() (resp. GLGraphics.setSceneCenter()) simply call this method (resp. setSceneCenter()) on its associated GLGraphics.camera(). Default value is (0,0,0) (world origin). Use setSceneCenter() to change it.

See Also:
setSceneBoundingBox().

setAspectRatio

public void setAspectRatio(float aspect)
Defines the GLCamera aspectRatio().

This value is actually inferred from the screenWidth() / screenHeight() ratio. You should use setScreenWidthAndHeight(int,int) instead.

This method might however be convenient when the GLCamera is not associated with a GLGraphcis. It actually sets the screenHeight() to 100 and the screenWidth() accordingly. See also setFOVToFitScene().

Note: If you absolutely need an aspectRatio() that does not correspond to your viewer's window dimensions, overload loadProjectionMatrix() or multiply the created GL_PROJECTION matrix by a scaled diagonal matrix in your GLGraphics.draw() method.

setFOVToFitScene

public void setFOVToFitScene()
Changes the GLCamera fieldOfView() so that the entire scene (defined by GLGraphics.sceneCenter() and GLGraphics.sceneRadius()) is visible from the GLCamera position().

The position() and orientation() of the GLCamera are not modified and you first have to orientate the GLCamera in order to actually see the scene (see lookAt(PVector), showEntireScene() or fitSphere(PVector,float)).

This method is especially useful for shadow maps computation. Use the GLCamera positioning tools (setPosition(PVector), lookAt(PVector)) to position a GLCamera at the light position. Then use this method to define the fieldOfView() so that the shadow map resolution is optimally used:

// The light camera needs size hints in order to optimize its fieldOfView
lightCamera.setSceneRadius(sceneRadius());
lightCamera.setSceneCenter(sceneCenter());
// Place the light camera
lightCamera.setPosition(lightFrame.position());
lightCamera.lookAt(sceneCenter());
lightCamera.setFOVToFitScene();
Attention: The fieldOfView() is clamped to M_PI/2.0. This happens when the GLCamera is at a distance lower than sqrt(2.0) * sceneRadius() from the sceneCenter(). It optimizes the shadow map resolution, although it may miss some parts of the scene.

setUpVector

public void setUpVector(PVector up,
                        boolean noMove)
Rotates the GLCamera so that its upVector() becomes up (defined in the world coordinate system).

The GLCamera is rotated around an axis orthogonal to up and to the current upVector() direction.

Use this method in order to define the GLCamera horizontal plane.

When noMove is set to false, the orientation modification is compensated by a translation, so that the revolveAroundPoint() stays projected at the same position on screen. This is especially useful when the GLCamera is an observer of the scene (default mouse binding).

When noMove is true, the GLCamera position() is left unchanged, which is an intuitive behavior when the GLCamera is in a walkthrough fly mode. See also setViewDirection(), lookAt() and setOrientation().

public final GLCamera.Type type()

Set by setType(GLCamera.Type). Mainly used by loadProjectionMatrix().

A Camera.PERSPECTIVE GLCamera uses a classical projection mainly defined by its fieldOfView().

unprojectedCoordinatesOf

public final PVector unprojectedCoordinatesOf(PVector src,
                                              GLFrame frame)
Returns the world unprojected coordinates of a point src defined in the screen coordinate system.

The src.x and src.y input values are expressed in pixels, (0,0) being the upper left corner of the window. src.z is a depth value ranging in [0..1[ (near and far plane respectively). See the gluUnProject man page for details.

The result is expressed in the frame coordinate system. When frame is null, the result is expressed in the world coordinates system. The possible frame GLFrame.referenceFrame() are taken into account.

projectedCoordinatesOf(PVector,GLFrame) performs the inverse transformation.

This method only uses the intrinsic GLCamera parameters (see getModelViewMatrix(), getProjectionMatrix() and getViewport()) and is completely independent of the OpenGL GL_MODELVIEW, GL_PROJECTION and viewport matrices. You can hence define a virtual GLCamera and use this method to compute un-projections out of a classical rendering context.

Attention: However, if your GLCamera is not attached to a GLGraphics (used for offscreen computations for instance), make sure the GLCamera matrices are updated before calling this method (use computeModelViewMatrix(), computeProjectionMatrix()). This method is not computationally optimized. If you call it several times with no change in the matrices, you should buffer the entire inverse projection matrix (modelview, projection and then viewport) to speed-up the queries. See the gluUnProject man page for details. See also setScreenWidthAndHeight().

zClippingCoefficient

public float zClippingCoefficient()
Returns the coefficient used to position the near and far clipping planes.

The near (resp. far) clipping plane is positioned at a distance equal to zClippingCoefficient() * sceneRadius() in front of (resp. behind) the sceneCenter(). This guarantees an optimal use of the z-buffer range and minimizes aliasing. See the zNear() and zFar() documentations.

Default value is square root of 3.0 (so that a cube of size sceneRadius() is not clipped).

However, since the sceneRadius() is used for other purposes (see showEntireScene(), flySpeed(), ...) and you may want to change this value to define more precisely the location of the clipping planes. See also zNearCoefficient().

zFar

public float zFar()
Returns the far clipping plane distance used by the Camera projection matrix.

The far clipping plane is positioned at a distance equal to zClippingCoefficient() * sceneRadius() behind the sceneCenter():

zFar = distanceToSceneCenter() + zClippingCoefficient()*sceneRadius();

See Also:
zNear()

zNear

public float zNear()
Returns the near clipping plane distance used by the GLCamera projection matrix.

The clipping planes' positions depend on the sceneRadius() and sceneCenter() rather than being fixed small-enough and large-enough values. A good scene dimension approximation will hence result in an optimal precision of the z-buffer.

The near clipping plane is positioned at a distance equal to zClippingCoefficient() * sceneRadius() in front of the sceneCenter(): distanceToSceneCenter() - zClippingCoefficient() * sceneRadius()

In order to prevent negative or too small zNear() values (which would degrade the z precision), zNearCoefficient() is used when the GLCamera is inside the sceneRadius() sphere:

zMin = zNearCoefficient() * zClippingCoefficient() * sceneRadius();
zNear = zMin;
// With an ORTHOGRAPHIC type, the value is simply clamped to 0.0

See also the zFar(), zClippingCoefficient() and zNearCoefficient() documentations.

If you need a completely different zNear computation, overload the zNear() and zFar() methods in a new class that publicly inherits from Camera and use GLGraphics.setCamera().

Attention: The value is always positive although the clipping plane is positioned at a negative z value in the Camera coordinate system. This follows the gluPerspective standard.

zNearCoefficient

public float zNearCoefficient()
Returns the coefficient which is used to set zNear() when the GLCamera is inside the sphere defined by sceneCenter() and zClippingCoefficient() * sceneRadius().

In that case, the zNear() value is set to zNearCoefficient() * zClippingCoefficient() * sceneRadius(). See the zNear() documentation for details.

Default value is 0.005, which is appropriate for most applications. In case you need a high dynamic ZBuffer precision, you can increase this value (~0.1). A lower value will prevent clipping of very close objects at the expense of a worst Z precision.

Only meaningful when GLCamera type is GLCamera.PERSPECTIVE.
