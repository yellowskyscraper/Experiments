Interface GLMouseGrabber

public interface GLMouseGrabber
Author:
pierre Interface for objects that grab mouse focus in a GLGraphics.

GLMouseGrabber are objects which react to the mouse cursor, usually when it hovers over them.

How does it work ?

All the created GLMouseGrabbers are grouped in a {@link #mouseGrabberPool()}. The GLGraphics parse this pool, calling all the GLMouseGrabbers' {@link #checkIfGrabsMouse(int, int, GLCamera)} methods that {@link #setGrabsMouse(boolean)} if desired (method calls should actually be performed on concrete class instances such as GLInteractiveFrame).

When a GLMouseGrabber {@link #grabsMouse()}, it becomes the GLGrahics.mouseGrabber(). All the mouse events are then transmitted to it instead of being normally processed. This continues while {@link #grabsMouse()} (updated using {@link #checkIfGrabsMouse(int, int, GLCamera)}) returns {@code true}.

If you want to (temporarily) disable a specific GLMouseGrabbers, you can remove it from this pool using {@link #removeFromMouseGrabberPool()}.

checkIfGrabsMouse

public void checkIfGrabsMouse(int x,
                              int y,
                              GLCamera camera)
Called by the GLGraphics before they test if the GLMouseGrabber grabsMouse(). Should setGrabsMouse(boolean) according to the mouse position.

This is the core method of the GLMouseGrabber. Its goal is to update the grabsMouse() flag according to the mouse and GLMouseGrabber current positions, using setGrabsMouse(boolean). grabsMouse() is usually set to true when the mouse cursor is close enough to the GLMouseGrabber position. It should also be set to false when the mouse cursor leaves this region in order to release the mouse focus. x and y are the mouse cursor coordinates ((0,0) corresponds to the upper left corner).

A typical implementation will look like:

// (posX,posY) is the position of the GLMouseGrabber on screen.
// Here, distance to mouse must be less than 10 pixels to activate the MouseGrabber.
setGrabsMouse( PApplet.sqrt((x-posX)*(x-posX) + (y-posY)*(y-posY)) < 10);

If the GLMouseGrabber position is defined in 3D, use the camera parameter, corresponding to the calling GLGraphics GLCamera. Project on screen and then compare the projected coordinates:

PVector proj = new PVector (camera.projectedCoordinatesOf(myGLMouseGrabber.frame().position());
setGrabsMouse((PApplet.abs(x-proj.x) < 5) && (PApplet.(y-proj.y) < 2)); // Rectangular region

getMouseGrabberPool

public List getMouseGrabberPool()
Returns a list containing references to all the active GLMouseGrabbers.

Used by the GLGraphics to parse all the GLMouseGrabbers and to check if any of them grabsMouse() checkIfGrabsMouse(int,int,GLCamera).

You should not have to directly use this list. Use removeFromMouseGrabberPool() and addInMouseGrabberPool() to modify this list.

mousePressEvent

public void mousePressEvent(MouseEvent event,
                            GLCamera camera)
Callback method called when the GLMouseGrabber grabsMouse() and a mouse button is pressed.

The GLMouseGrabber will typically start an action or change its state when a mouse button is pressed. mouseMoveEvent(MouseEvent,GLCamera) (called at each mouse displacement) will then update the GLMouseGrabber accordingly and mouseReleaseEvent(MouseEvent,GLCamera) (called when the mouse button is released) will terminate this action.
