A GLFrame is a 3D coordinate system, represented by a position() and an orientation(). The order of these transformations is important: the GLFrame is first translated and then rotated around the new translated origin.

alignWithFrame

public final void alignWithFrame(GLFrame frame,
                                 boolean move,
                                 float threshold)
Aligns the GLFrame with frame, so that two of their axis are parallel.

If one of the X, Y and Z axis of the GLFrame is almost parallel to any of the X, Y, or Z axis of frame, the GLFrame is rotated so that these two axis actually become parallel.

If, after this first rotation, two other axis are also almost parallel, a second alignment is performed. The two frames then have identical orientations, up to 90 degrees rotations.

threshold measures how close two axis must be to be considered parallel. It is compared with the absolute values of the dot product of the normalized axis.

When move is set to true, the GLFrame position() is also affected by the alignment. The new GLFrame position() is such that the frame frame position (computed with coordinatesOf(PVector), in the GLFrame coordinates system) does not change.

frame may be null and then represents the world coordinate system (same convention than for the referenceFrame()).

fromArray

public final void fromArray(float[] m)
Sets the GLFrame from an OpenGL matrix representation (rotation in the upper left 3x3 matrix and translation on the last line).

Hence, if a code fragment looks like:

float [] m = new float [16]; m[0]=...;
gl.glMultMatrixf(m);

It is equivalent to write:

GLFrame fr = new GLFrame();
fr.setFromMatrix(m);
gl.glMultMatrixf(fr.matrix());

Using this conversion, you can benefit from the powerful GLFrame transformation methods to translate points and vectors to and from the GLFrame coordinate system to any other GLFrame coordinate system (including the world coordinate system). See coordinatesOf() and transformOf().

Attention: A GLFrame does not contain a scale factor. The possible scaling in m will not be converted into the GLFrame by this method.

inverse

public final GLFrame inverse()
Returns a GLFrame representing the inverse of the GLFrame space transformation.

The rotation() the new GLFrame is the GLQuaternion.inverse() of the original rotation. Its translation() is the negated inverse rotated image of the original translation.

If a GLFrame is considered as a space rigid transformation (translation and rotation), the inverse() GLFrame performs the inverse transformation.

Only the local GLFrame transformation (i.e., defined with respect to the referenceFrame()) is inverted. Use worldInverse() for a global inverse.

The resulting GLFrame has the same referenceFrame() as the GLFrame and a null constraint().

Note: The scaling factor of the 4x4 matrix is 1.0.

matrix

public final float[][] matrix()
Returns the 4x4 OpenGL transformation matrix represented by the GLFrame.

This method should be used in conjunction with glMultMatrixf() to modify the OpenGL modelview matrix from a GLFrame hierarchy. For example with this GLFrame hierarchy:

GLFrame body = new GLFrame();
GLFrame leftArm = new GLFrame();
GLFrame rightArm = new GLFrame();
leftArm.setReferenceFrame(body);
rightArm.setReferenceFrame(body);

The associated OpenGL drawing code should look like:

gl.glPushMatrix();
gl.glMultMatrixd(body.matrix());
drawBody();
gl.glPushMatrix();
gl.glMultMatrixd(leftArm.matrix());
gl.drawArm();
gl.glPopMatrix();
gl.glPushMatrix();
gl.glMultMatrixd(rightArm.matrix());
drawArm();
gl.glPopMatrix();
gl.glPopMatrix();

Note the use of nested gl.glPushMatrix() and gl.glPopMatrix() blocks to represent the frame hierarchy: leftArm and rightArm are both correctly drawn with respect to the body coordinate system.

This matrix only represents the local GLFrame transformation (i.e., with respect to the referenceFrame()). Use worldMatrix() to get the full GLFrame transformation matrix (i.e., from the world to the Frame coordinate system). These two match when the referenceFrame() is null.

The result is only valid until the next call to matrix(), array(), worldMatrix() or worldArray(). Use it immediately (as above).

Attention: The OpenGL format of the result is the transpose of the actual mathematical European representation (translation is on the last line instead of the last column).

Note: The scaling factor of the 4x4 matrix is 1.0.

referenceFrame

public final GLFrame referenceFrame()
Returns the reference GLFrame, in which coordinates system the GLFrame is defined.

The translation() rotation() of the GLFrame are defined with respect to the reference GLFrame coordinate system. A null reference GLFrame (default value) means that the GLFrame is defined in the world coordinate system.

Use position() and orientation() to recursively convert values along the reference GLFrame chain and to get values expressed in the world coordinate system. The values match when the reference GLFrame null.

Use setReferenceFrame(GLFrame) to set this value and create a Frame hierarchy. Convenient functions allow you to convert 3D coordinates from one GLFrame to another: see coordinatesOf(PVector), localCoordinatesOf(PVector) , coordinatesOfIn(PVector,GLFrame) and their inverse functions.

Vectors can also be converted using transformOf(PVector), transformOfIn(PVector,GLFrame), localTransformOf(PVector) and their inverse functions.

worldInverse

public final GLFrame worldInverse()
Returns the inverse() of the GLFrame world transformation.

The orientation() of the new GLFrame is the GLQuaternion.inverse() of the original orientation. Its position() is the negated and inverse rotated image of the original position.

The result GLFrame has a null referenceFrame() and a null constraint().

Use inverse() for a local (i.e., with respect to referenceFrame()) transformation inverse.

worldMatrix

public final float[][] worldMatrix()
Returns the 4x4 OpenGL transformation matrix represented by the GLFrame.

This method should be used in conjunction with glMultMatrixf() to modify the OpenGL modelview matrix from a GLFrame:

// The modelview here corresponds to the world coordinate system. GLFrame fr = new GLFrame(pos, GLQuaternion(from, to));
gl.glPushMatrix();
gl.glMultMatrixf(fr.worldMatrix());
// draw object in the fr coordinate system.
glPopMatrix();

This matrix represents the global GLFrame transformation: the entire referenceFrame() hierarchy is taken into account to define the GLFrame transformation from the world coordinate system. Use matrix() to get the local GLFrame transformation matrix (i.e. defined with respect to the referenceFrame()). These two match when the referenceFrame() is null.

The OpenGL format of the result is the transpose of the actual mathematical European representation (translation is on the last line instead of the last column.

Attention: The result is only valid until the next call to matrix(), array() worldMatrix() or worldArray(). Use it immediately (as above).

ANote: The scaling factor of the 4x4 matrix is 1.0.
